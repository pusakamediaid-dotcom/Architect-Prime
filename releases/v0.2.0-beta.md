# Architect-Prime v0.2.0-beta

This beta release establishes the first production-style baseline for Architect-Prime while preserving the broader multi-architecture and multi-language roadmap.

## Highlights

- Replaced in-memory Node.js storage with real Prisma persistence.
- SQLite is the default zero-configuration database for local development.
- Added PostgreSQL-compatible Prisma schema for production-oriented deployments.
- Added repository layer to separate database access from services and controllers.
- Added Prisma migration and seed workflow.
- Added integration, unit, and API smoke tests.
- Added coverage reporting with an 80%+ line coverage target.
- Updated CI to run npm ci, Prisma generate, Prisma migrate, build, lint, test, coverage, and npm audit.
- Added validation screenshots under `assets/validation/`.
- Added `KNOWN_LIMITATIONS.md` for transparent beta limitations.

## Beta Limitations

- PostgreSQL support is provided via `prisma/schema.postgresql.prisma` because Prisma datasource providers are selected by schema, not only by URL.
- Go, PHP, Firebase, AWS Serverless, and Microservices modules are still preview/scaffold modules.
- Production deployment has not yet been fully benchmarked or validated end-to-end.

See [`KNOWN_LIMITATIONS.md`](../KNOWN_LIMITATIONS.md) for details.
